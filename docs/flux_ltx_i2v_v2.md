# FLUX→LTX 图生视频 v2 (`flux_ltx_i2v_v2.json`) ★ 推荐

> 两段话 → 3 秒高质量视频。FLUX 生成首帧 → LTX 生成运动。33 节点，需要 22-24GB 显存。

## 流程图

阶段 1: FLUX.1-DEV 文生图 (节点 1-11)
    └→ 根据文本 Prompt 生成一张高质量的初始帧图片 (960×544)

阶段 2: LTX Video 2.3 图生视频 (节点 12-33)
    └→ 以 FLUX 生成的图片为起点，根据运动 Prompt 生成 73 帧 (≈3秒) 连贯视频

具体可参考 flux_ltx_i2v_v2_文档.md

## 工作流程

```
阶段1: FLUX.1-DEV 文生图 (节点 1-11)
  └→ 根据 Prompt 生成首帧画面

阶段2: FLUX→LTX 桥接 (节点 10-20)
  └→ FLUX VAE 解码 → 像素 → LTX 预处理 → LTX VAE 编码

阶段3: LTX Video 2.3 视频生成 (节点 12-33)
  └→ 根据运动 Prompt 生成 73 帧连贯视频
```

## 怎么用

1. 拖入 `flux_ltx_i2v_v2.json`
2. **节点 5**（FLUX Prompt）：描述**首帧画面**
3. **节点 13**（LTX Prompt）：描述**接下来的运动**
4. 节点 8：分辨率（默认 960×544）
5. 节点 21：帧数（73 帧 ≈ 3 秒 @ 24fps）
6. 点击 Queue Prompt

## Prompt 写法

```
✅ FLUX: "哑光黑超跑斜停在湿漉漉的沿海公路，金色夕阳逆光，低角度，静止，无人"
        ↑ 只描述"这一帧的画面"

✅ LTX:  "引擎咆哮，后轮咬住湿沥青猛烈弹射，水雾从轮拱喷出，尾灯拉成红光"
        ↑ 只描述"接下来发生的运动"

❌ 两个 Prompt 写一样的内容 ← 浪费
```

## v2 参数（与基线版对比）

| 参数 | 基线版 | v2 ★ | 说明 |
|------|--------|------|------|
| T5 编码器 | FP16 (9.2GB) | **FP8 (4.6GB)** | 省显存 |
| NAG scale | 12 | **10** | 减弱引导防伪影 |
| NAG tau | 2.5 | **3.5** | 增强时间平滑 |
| I2V strength | 0.80 | **0.85** | 紧锚定首帧 |
| 引导方式 | CFG | **STG 1.0** | 时空跳跃引导 |
| 步数 | 25 | **30** | 更多去噪 |
| terminal | 0.1 | **0.05** | 更干净 |

## 关键参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| FLUX steps | 28 | 首帧采样 |
| LTX steps | 30 | 视频采样，越高运动越细腻 |
| I2V strength | 0.85 | 首帧锚定。0.80-0.90 最佳 |
| NAG scale | 10 | 运动引导。8-14 安全 |
| NAG tau | 3.5 | 时间平滑。2.0-4.0 |
| STG stg | 1.0 | 时空引导。1.0=标准 |
| frame_rate | 24 | 帧率 |
| 帧数 | 73 | ≈3 秒 |

## 三版本对比

| | 基线版 | 优化版 | v2 ★ |
|------|--------|--------|------|
| 节点 | 32 | 33 | 33 |
| 文件 | `flux_ltx_i2v.json` | `flux_ltx_i2v_optimized.json` | `flux_ltx_i2v_v2.json` |
| 引导 | CFG | STG 1.5 | STG 1.0 |
| 稳定性 | 中等 | 差 | **最好** |
| 推荐 | 📦 存档 | ⚠️ 实验 | ✅ **当前** |

## 模型依赖

- FLUX 全套：flux1-dev + clip_l + t5xxl_fp8 + ae
- LTX 全套：UNet + VAE + Gemma + 投影层 + LoRA
